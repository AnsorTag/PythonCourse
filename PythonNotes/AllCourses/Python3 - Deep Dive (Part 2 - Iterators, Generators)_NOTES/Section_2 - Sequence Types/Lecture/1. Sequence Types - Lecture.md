## ğŸ”— Concatenation & Repetition Caveats

- **Concatenation** of sequences creates a **new object**.
  - **Watch out for mutable objects!** If a sequence contains mutable elements (e.g., lists), the same object can appear multiple times, causing unexpected mutations ğŸ”„.
  - Example:
    ```python
    x = [[0, 0]]
    a = x + x  # [[0, 0], [0, 0]] (same object twice)
    a[0][0] = 100  # Mutates both! [[100, 0], [100, 0]]
    ```

- **Repetition** has the same issue:
  - For immutable types (e.g., strings, ints), it's **safe**:
    ```python
    a = [1, 2] * 2  # [1, 2, 1, 2]
    ```
  - For mutable types, the repeated items are **the same object**:
    ```python
    x = [[0, 0]]
    a = x * 2  # [[0, 0], [0, 0]]
    a[0][0] = 100  # Mutates both! [[100, 0], [100, 0]]
    ```
